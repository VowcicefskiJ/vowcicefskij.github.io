Project Description

Original Project: CS 360 Basic Inventory Management App
When I Built It: Earlier in 2024
My Enhancement Goal: Learn professional software architecture and security practices

This project started as a simple Android app for my CS 360 course where users could track inventory items and get notifications when stock was low. The original version worked, but it had some major problems that I wanted to fix. The business logic was mixed in with the UI code, passwords were stored as plain text, and everything was tightly coupled together, making it hard to maintain or test.

Why I Chose This Project

I selected this project because it gave me a chance to transform functional student code into something that follows professional standards. The original app had architectural and security issues that I knew I could improve using concepts I'd learned throughout the program.

Learning MVVM Architecture

I implemented the Model-View-ViewModel pattern to separate concerns properly. This was challenging but taught me how professional Android apps are structured.

Creating Modular Code

I broke my code into utility classes like SecurityUtils and ValidationUtils. This eliminated code duplication and made testing much easier.

Implementing Real Security

I replaced plain-text passwords with proper SHA-256 hashing and added input sanitization to prevent common attacks.

Improving Performance

I learned how proper architecture actually makes apps run better by managing state more efficiently and reducing unnecessary operations.

How This Project Meets Course Outcomes

  • Collaborative Environments: My modular architecture makes it easy for other developers to work on different parts without conflicts
  • Professional Communication: I documented my code clearly and can explain my architectural decisions to both technical and non-technical audiences
  • Industry-Standard Techniques: I learned and applied current Android development practices like MVVM and Jetpack Compose
  • Security Mindset: I implemented cryptographic security measures and defensive programming practices throughout the app

Implementing Security Best Practices

Learning about cryptographic security was eye-opening:

// My implementation of secure password handling
object SecurityUtils {
    fun hashPasswordWithSalt(password: String, salt: ByteArray? = null): Pair<String, ByteArray> {
        val actualSalt = salt ?: generateSalt()
        val digest = MessageDigest.getInstance("SHA-256")
        
        digest.update(actualSalt)
        val hashedBytes = digest.digest(password.toByteArray(Charsets.UTF_8))
        val hashedPassword = hashedBytes.joinToString("") { "%02x".format(it) }
        
        return Pair(hashedPassword, actualSalt)
    }
    
    private fun constantTimeEquals(a: String, b: String): Boolean {
        if (a.length != b.length) return false
        var result = 0
        for (i in a.indices) {
            result = result or (a[i].code xor b[i].code)
        }
        return result == 0
    }
}

Creating Reusable Utility Classes

I learned how to organize code to avoid repetition:

// My approach to input validation
object ValidationUtils {
    data class ValidationResult(val isValid: Boolean, val errorMessage: String = "")
    
    fun validateLoginInput(username: String, password: String): ValidationResult {
        if (username.isBlank() || password.isBlank()) {
            return ValidationResult(false, "Please fill in all fields")
        }
        
        if (username.trim().length < 3) {
            return ValidationResult(false, "Username must be at least 3 characters")
        }
        
        return ValidationResult(true)
    }
}
